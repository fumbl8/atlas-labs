# Public Boundaries

Atlas Labs is intentionally presented as an early public portfolio, not as a completed organization or product suite.

## Claims intentionally avoided

This public version does not claim that Atlas Labs is:

- a formal company,
- a funded laboratory,
- a clinical tool developer,
- a conservation-management authority,
- a validated scientific research program,
- a deployed software platform,
- or an institution with users, pilots, customers, or field deployments.

## What is fair to claim

It is fair to describe Atlas Labs as:

- an independent AI/systems portfolio,
- a biology/ecology/computational-systems workbench,
- a collection of WIP prototypes and research-navigation concepts,
- a local-first agent-systems exploration layer,
- and a public index of early proof artifacts.

## Validation posture

Projects are labeled according to their current maturity:

- **Concept** — idea, architecture, prompt, or schema without a complete implementation.
- **Prototype** — runnable or partially runnable artifact, usually narrow and locally scoped.
- **Learning notebook** — educational or self-training artifact, not a validated method.
- **Case study** — bounded example that may use public data but still requires review and validation.

## Sensitive material removed from public version

The public repo intentionally excludes:

- private planning notes,
- local machine paths,
- personal operating handoffs,
- internal automation state,
- imported-source metadata,
- raw scratch outputs,
- cache folders,
- private project-history files,
- and anything that looked like system-internal scaffolding rather than public portfolio material.
